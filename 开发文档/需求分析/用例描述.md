# 用例描述
```
用例名称:管理科室
用例编号:UC19
用例简介:管理员修改线上预约系统的所有科室及其介绍信息，并在网站上显示
参与者:管理员
触发条件:管理员准备修改科室信息
前置条件:管理员登录
基本流程：
    (用户) 1. 管理员登录管理系统。
    (系统) 2. 系统检测账号密码。
    (用户) 3. 管理员修改线上预约系统的所有科室及其介绍信息。
    (系统) 4. 系统显示最新信息。
扩展流程：
    (可选) 3-4a. 管理员可以在任何时间选择退出系统。
        (系统)1. 系统进入管理员模式。
        (系统)2. 管理员执行退出操作。
        (用户)3. 系统退出，保存当前信息。
    (异常处理) 2a. 当用户登录时账号或密码错误。
         (系统)1. 系统提示用户账号密码错误。
         (用户)2. 用户再次输入账号密码。
         (系统)3. 系统再次检测用户账号密码。
结论：弹出修改信息成功弹窗
后置条件：系统信息修改，在网站上可查询
补充约束：
    1. 系统响应迅速，登录响应在1s之内。
    2. 3-4中能够加入可插拔的业务规则。
    3. UI清晰，使用户便于使用。
```
