## 用户故事

后台可以管理线上预约系统的所有科室及其介绍信息，每个科室的所有医生及其介绍信息。后台还能设置每个医生的出诊排班表（例如只有周一、三、五上班，只有上午上班等）以及放号数量（例如上午放十个号，或者上午每二十分钟作为一个时间段放号等）。

* 用户故事1：作为一个后台管理员，我想要管理线上预约系统的所有科室及其介绍信息，以便于患者选择对应的科室就医

* 用户故事2：作为一个后台管理员，我想要管理每个科室的所有医生及其介绍信息，以便于患者选择对应的医生就医

* 用户故事3：作为一个后台管理员，我想要设置每个医生的出诊排班表，以便医生能够了解自己的上班时间与患者安排就医时间

* 用户故事4：作为一个后台管理员，我想要设置放号数量，以便减少医生的压力与维持医院的秩序

## 原始自然语言需求

- 管理员在登录的状态下，能够进行管理员的操作（ST）
- 在管理员登录的情况下，管理员可以修改线上预约系统的所有科室及其介绍信息（UB）
- 在登录的情况下，管理员可以修改放号数量。（UB）
- 在登录的情况下，管理员可以设置每个医生的出诊排班表（UB）

- 如果医生有修改排班表需求时，则管理员可以登陆管理系统修改医生的出诊排班表（UW）

## 基本流程

- 用户故事1：作为一个后台管理员，我想要管理线上预约系统的所有科室及其介绍信息，以便于患者选择对应的科室就医

```
作为一个后台管理员，我想要管理线上预约系统的所有科室及其介绍信息，以便于患者选择对应的科室就医
 基本流程{
    (用户) 1. 管理员登录管理系统。
    (系统) 2. 系统检测账号密码。
    (用户) 3. 管理员修改线上预约系统的所有科室及其介绍信息。
    (系统) 4. 系统显示最新信息。
}
扩展流程{
    (可选) 3-4a. 管理员可以在任何时间选择退出系统。
        (系统)1. 系统进入管理员模式。
        (系统)2. 管理员执行退出操作。
        (用户)3. 系统退出，保存当前信息。
    (异常处理) 2a. 当用户登录时账号或密码错误
         (系统)1. 系统提示用户账号密码错误
         (用户)2. 用户再次输入账号密码
         (系统)3. 系统再次检测用户账号密码
}   
非功能需求：{
1. 系统响应迅速，登录响应在1s之内。
2. 3-4中能够加入可插拔的业务规则。
3. UI清晰，使用户便于使用。
}
```
